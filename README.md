## 経過
どのようなRDBMSを用いるか  
### -- MySQL --  

<利点><br>
- 利用可能データ量が膨大
- 複数同時アクセスに強い
- 各個人に権限付与可能
<欠点><br>
- サーバー・環境構築が必須

### -- SQLite --  

<利点><br>
- 環境構築・サーバーが不要
<欠点><br>
- 複数同時アクセスに弱い(特に書き込み)
- 権限設定が不可能
- 利用可能データ量が少ない
